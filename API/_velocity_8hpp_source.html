112  constexpr inline Velocity operator/ (const Velocity& lhs, const Real rhs)
113  {
114  /*
115  * While it can be argued that division operations shouldn't be supported due to
116  * hardware support for division typically being significantly slower than hardware
117  * support for multiplication, it can also be argued that it shouldn't be the
118  * software developer's role to attempt to optimize what the compiler should be
119  * much better at knowing how to optimize. So here the code chooses the latter
120  * strategy which allows the intention to be clearer, and just passes the division
121  * on down to the Vec2 and Angle types (rather than manually rewriting the divisions
122  * as multiplications).
123  */
124  return Velocity{lhs.linear / rhs, lhs.angular / rhs};
